## 🚫 核心痛点
* **现象**：逻辑都懂，但漏了一个 `printf`，或者用了旧的变量值，或者跳过了一行代码。
* **原因**：**过度自信的“脑补”**。大脑喜欢偷懒，自动跳过繁琐步骤。
* **对策**：**降智打击**。把自己当成没有任何感情的 CPU，严格单步执行。

---

## 🛠️ 三大防呆机制

### 1. 防“漏输出”：圈地运动 (Circle & Box)
> **针对错误**：漏掉循环外的汇总输出，或漏掉换行符。

* **操作步骤**：
    1.  **不动笔算之前**，先扫描全篇代码。
    2.  用笔**圈出所有的 `printf`** (包括 `printf("\n")`)。
    3.  在草稿纸显眼处画一个方框，标注为 **【屏幕输出区】**。
    4.  **执行原则**：只有代码运行到画圈处，才允许往方框里写字。除此之外，不要在方框里写任何计算过程。

### 2. 防“变量记混”：变量追踪表 (Trace Table)
> **针对错误**：数组被修改了 3 次，你却还在用第 1 次的值计算（如数组翻转题、字符串修改题）。

* **操作步骤**：
    1.  **禁止心算**！在草稿纸上画表。
    2.  **表头**：列出所有关键变量名 (`i`, `n`, `s1`, `Array[]`)。
    3.  **更新规则**：
        * 变量值一变，**立刻划掉旧值**，在旁边写上新值。
        * 计算下一步时，**强制眼球去看表格里的“最新值”**，绝对不要看卷子上印的初始值。

    **表格示例**：
    ```text
    Step          | s1 (字符串)   | n (计数) | i (循环)
    --------------|---------------|----------|----------
    Init          | "awCve"       | 0        | 0
    Loop 1        |               | 0 -> 1   | 0 -> 1
    s1 Modified   | "awCve" -> "X"|          |
    ```

### 3. 防“漏执行”：指读法 (Finger Tracing)
> **针对错误**：看太快，直接跳过了某句 `func()` 调用，导致变量没更新。

* **操作步骤**：
    1.  **笔尖定位**：用笔尖指着当前执行的这一行代码。
    2.  **物理标记**：执行完一行，在行尾**打个勾**或者**划一条横线**。
    3.  **函数进入**：遇到 `func(...)`，必须在草稿纸上模拟“进入”函数，不要试图直接猜结果。

## 🚀 考场实战口诀

1.  **输出先画圈**，草稿留专区。
2.  **变量列成表**，旧值必划掉。
3.  **指读不跳行**，执行打个勾。
4.  **我是笨 CPU**，莫得感情莫得脑补。
