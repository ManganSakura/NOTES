# 梯队培训第一讲：环境配置与基础工具

> [!abstract] 课程摘要
> **主讲内容**：开发环境搭建、Linux 基础指令、工具链使用。
> **目标**：完成系统配置，能够独立编译运行第一个程序。

## 1. 开发环境搭建 (Environment Setup)

工欲善其事，必先利其器。本节主要介绍操作系统的安装与配置。

### 1.1 系统选择
* **操作系统**：Ubuntu 20.04 / 22.04 LTS (推荐)
* **虚拟化方案**：
    * [ ] **WSL2** (Windows Subsystem for Linux)：轻量级，适合日常开发，与 Windows 交互方便。
    * [ ] **VMware / VirtualBox**：传统的虚拟机，GUI 支持较好，但资源占用高。
    * [ ] **双系统 (Dual Boot)**：性能最好，适合跑大型仿真 (Gazebo) 和实机调试。

> [!tip] 建议
> 初学者推荐使用 **WSL2** 或 **双系统**。WSL2 配置最快，双系统最稳定。

### 1.2 必备软件清单

| 软件名称                  | 用途    | 推荐理由                         |
| :-------------------- | :---- | :--------------------------- |
| **Terminator**        | 终端模拟器 | 支持分屏，ROS 调试神器                |
| **VS Code**           | 代码编辑器 | 插件丰富，支持远程开发 (Remote-SSH/WSL) |
| **Git**               | 版本控制  | 团队协作必备                       |
| **Typora / Obsidian** | 笔记工具  | Markdown 实时预览，知识管理           |

## 2. Linux 基础指令 (Basic Commands)

> [!warning] 注意
> 在 Linux 中，**文件路径** 和 **文件名** 是区分大小写的！

### 常用文件操作
```bash
pwd                   # 显示当前路径 (Print Working Directory)
ls -al                # 列出所有文件 (包括隐藏文件)
cd /path/to/dir       # 切换目录 (Change Directory)
mkdir build           # 创建文件夹 (Make Directory)
rm -rf folder_name    # 强制删除文件夹 (慎用!)
touch main.c          # 创建空文件
````

### 权限与软件安装

Bash

```
sudo apt update       # 更新软件源列表
sudo apt install git  # 安装软件 (以 git 为例)
chmod +x script.sh    # 赋予执行权限
```

## 3. 编译工具链 (Build Tools)

### GCC 与 CMake

在 Linux 下，我们通常使用 `gcc/g++` 编译 C/C++ 代码，用 `CMake` 来管理构建过程。

Code snippet

```
graph LR
    A[源代码 .c/.cpp] -->|预处理| B[预编译文件 .i]
    B -->|编译| C[汇编代码 .s]
    C -->|汇编| D[目标文件 .o]
    D -->|链接 (Linker)| E[可执行文件]
```

### 编译演示 (Hello World)

1. **编写代码** `vim hello.c`
    
2. **编译**：
    
    Bash
    
    ```
    gcc hello.c -o hello
    ```
    
3. **运行**：
    
    Bash
    
    ```
    ./hello
    ```
